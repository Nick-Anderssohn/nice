## 1. Executive verdict

**Path B still stands. No verifier, tiebreak, defend, or gap flipped it — every verdict's impact assessment explicitly reads "does not flip/overturn Path B." But the *basis* of confidence changes materially: §12's "no longer provisional" (:483) is refuted, and Path B should revert to *provisional* pending a short spike program the report skipped.**

What survives adversarial re-derivation (the load-bearing pillars):

- **The live FPS gate is real.** The single-stack GPUI-native terminal's locked-60 run recomputes exactly from the on-disk CSV (p50 16.666 / p95 16.813 / p99 17.670 ms, 1078 frames/17.99 s, debug build) — and the comparison bias ran *against* B (it ingested ~2× Path A's bytes on a ~2.3×-cell grid and still locked cadence).

- **Path A's structural liabilities are real:** txn's p95 ~31 ms tail re-derives byte-for-byte from raw artifacts and exists only via a fork patch (commit 583551f, off by default); the two-toolchain @_cdecl/reverse-FFI is structural; the hitTest-swizzle seam is real (though PoC-proven workable, 6/6 deterministic).

What does not survive:

1. **The §11 "three independent lenses converged at 9.0" framing should carry near-zero weight** (C01). All three lenses shared one evidence corpus and a quality-primary objective; two of three prompts pre-encoded Path A's exact liabilities as the rubric; and Path B (9,9,9) row-dominates every alternative, so no lens weighting could ever have changed rank #1 (gap G16). The decision rests on the §10/§12 measured + source-verified evidence — which still favors B — not on the table.
2. **Key absolutes downgrade to conditionals** (C03/C05/C11/C19): "no fork" holds only if Nick accepts plain-coverage text and integer-pixel vertical scroll steps on GPUI main, or budgets small **additive** GPUI patches for Nice's shipping stem-darkened look and sub-pixel scroll — the very fork class the decision rule treats as the Path A trigger, at trivial scale. The honest wording is §11's own "narrow and conditional."
3. **Four recommendation-threatening gaps must close before commitment** (§5): the IME gate was closed on a demonstrably broken existence proof (and the arbitration policy lives in GPUI's platform key dispatch, which app code cannot override); the AA/gamma gate was closed with zero rendered pixels and the static reading is wrong in places; the founding "Claude weak at Swift, strong at Rust" premise is unmeasured and eroded by mid-2026 facts (Xcode 26.3 ships native Claude Agent SDK integration); and the shipping app was scored 4.5/10 on quality without naming a single user-visible defect — under the stated quality-max criteria the null hypothesis was mis-scored, not out-ranked.
4. **Two never-considered blockers:** Zed's `terminal`/`terminal_view` crates are **GPL-3.0-or-later** while Nice is MIT — ":410 ADOPT AS STARTING POINT" is not legally available as code (found independently by three gap lenses); and most of a terminal's *input/encoding* surface (mouse-report encoders, bracketed paste, OSC 52, **OSC 7 cwd tracking — a shipping Nice feature silently dropped by alacritty_terminal**, momentum-scroll filtering, kitty end-to-end) is unbudgeted in Path B's cost list.

Several audit findings actively **strengthen** B: selection-across-eviction resolves fork-free in vanilla alacritty (defend:C25); Zed's alacritty fork's VT core is byte-identical to upstream 0.26.0 (tty-only delta); a public `Application::with_platform` renderer seam exists on main (tiebreak:C12); and kitty encoders exist in MIT/Apache crates the survey missed.

**Net: direction unchanged, margin narrower than claimed. The honest state is "Path B, provisional again" — commit only after the ranked spikes in §6 (led by the AA pixel A/B and the IME live spike) come back clean, with Path A retained as a live fallback rather than a retired one.**
