Nice · Restyle

Transparent, unified-surface window mocks

Four variants of the same window: two tab styles × two titlebar heights, rendered with the shipped tokens — niceDefaultDark terminal theme, NICE_DARK chrome inks, Terracotta accent. The sidebar is flat (terminal font, colors, and background; active session in accent text; 1px hairline at its edge). No titlebar fill, no separator rule — one translucent surface, ~80% opaque with window blur. Tab two on every strip is deliberately over-long to exercise truncation. The Scheme toggle swaps every window to NICE_LIGHT + niceDefaultLight (hairlines flip to ink-alpha). Sidebar controls live in their decided homes: collapse toggle in the titlebar beside the traffic lights; tabs/files mode switcher and the gear in the sidebar footer. Status dots use the real StatusDot tokens — idle = ink3, thinking = brand Terracotta, waiting = blue — shown static here; the app keeps its ring + breathe pulse animations. 2026-07-15 feel-check round: the Inactive tabs control compares three candidate affordances for making unselected tabs read as clickable (grey underline / hairline dividers / faint rest chip); the Tabs control previews the single-session state — one tab renders as the window's centered titlebar text, no tab chrome, no overflow chevron; and the footer gear is now a real stroke cog (source-of-record geometry; the shipped sun-like gear is superseded).

Style A

Text + accent underline

Tab titles are plain text — inactive tabs in tertiary ink, the active tab in full ink with a 2px Terracotta underline seated on the bar's bottom edge. A truncated title reads as text, not a cramped box. The ✕ is shown on the active tab the way it would appear on hover.

Style B

Full-height divided tabs

The native macOS titlebar-tabs shape: each tab spans the bar's full height, separated by 1px hairlines, and the active tab carries a faint translucent fill. More visible structure than Style A — the tab region reads as a control strip rather than floating labels.

What changes under the hood

Chrome geometry

  • TOP_BAR_HEIGHT 52 → 28 or 38; traffic-light placer follows (native default at 28).
  • Card constants (CARD_INSET, radius, border, shadow) retire with the floating sidebar.
  • Titlebar band + 1px line rule removed — no fill from window top.

Sidebar

  • Same surface as the terminal — no separate fill, so transparency is uniform.
  • Typography switches to NICE_MONO at terminal size.
  • Active row: accent-colored text, no fill; hairline at the right edge, starting below the bar.

Transparency

  • Terminal + chrome background alpha from a new opacity slider (default ~80%).
  • WindowBackgroundAppearance::Blurred; blur radius via a small vendored-gpui patch to unlock a slider.
  • Truncation: tabs get a max width and ellipsize; full title on hover tooltip.
